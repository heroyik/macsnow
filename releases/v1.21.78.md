## Release Body

MacSnow 1.21.78 is a stability-focused release for the macOS menu bar prototype. It hardens process ownership so MacSnow cannot leave multiple menu bar instances behind, and it makes release builds the only supported build path.

### What's Changed

- Enforced single-instance runtime behavior:
  - Starting MacSnow while it is already running now terminates the existing process first.
  - The newly launched process then acquires the instance lock and continues as the only running MacSnow process.
  - If the existing process does not exit after `SIGTERM`, MacSnow escalates to `SIGKILL`.
  - If the lock still cannot be acquired, the new process exits instead of allowing two instances.
- Blocked accidental debug builds:
  - Default `swift build` debug builds now fail intentionally.
  - Release-only build commands are documented and enforced.
  - Debug build sentinel files prevent `.build/.../debug` directories from being recreated in this workspace.
- Improved packaging hygiene:
  - The app bundle now includes `CFBundleDisplayName`.
  - DMG staging apps are unregistered from LaunchServices during cleanup to avoid stale menu bar entries.
- Updated version metadata to `1.21.78`.

### Install

1. Download `MacSnow-1.21.78.dmg` from this release.
2. Open the DMG.
3. Drag `MacSnow.app` to the `/Applications` shortcut.
4. Launch `MacSnow.app` from Finder or Spotlight.

Because this is still an unsigned prototype, macOS may block the first launch. If that happens, open `System Settings` -> `Privacy & Security`, find the blocked `MacSnow.app` message, and choose `Open Anyway`. You can also Control-click `MacSnow.app`, choose `Open`, then confirm the launch prompt.

### Update Notes

If an older MacSnow instance is running, launching this version will terminate the older process and start the new one. This is intentional and prevents duplicate MacSnow entries in the menu bar settings.

### Known Limitations

- MacSnow is unsigned and not notarized.
- There is no auto-update mechanism yet.
- Manual install through the DMG is still required.
- Automated test targets are not yet available.
