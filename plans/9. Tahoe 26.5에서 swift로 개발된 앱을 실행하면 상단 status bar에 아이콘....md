# macOS Tahoe 26.5 Status Bar 아이콘 미표시 진단 및 실행 계획

## 목적

MacSnow가 macOS Tahoe 26.x에서 실행될 때 상단 메뉴 막대(Status Bar/Menu Bar)에 아이콘이 보이지 않는 문제를 재현 가능한 절차로 진단하고, 검증된 조치부터 순서대로 적용한다.

이 문서는 원인을 단정하지 않는다. Apple 문서로 확인된 동작, 현재 코드 상태, 아직 검증이 필요한 가설을 분리해서 관리한다.

## 확인된 사실

- macOS Tahoe 26의 시스템 설정에는 `Menu Bar` 설정이 있으며, `Allow in the Menu Bar` 섹션에서 일부 앱의 메뉴 막대 항목 표시 여부를 선택할 수 있다.
- Apple의 `NSStatusItem` API는 `NSStatusBar.system.statusItem(withLength:)`로 status item을 생성하고 메뉴 막대에 추가하는 AppKit 경로다.
- MacSnow는 현재 앱 번들에서 `LSUIElement`를 `true`로 설정한다.
- MacSnow는 `main.swift`에서 `NSApplication` activation policy를 `.accessory`로 설정한다.
- MacSnow는 `StatusMenuController`에서 `NSStatusItem`을 유지하고, `autosaveName`을 `local.macsnow.statusItem`으로 설정한다.
- MacSnow는 status item 버튼에 template SF Symbol 기반 눈송이 이미지를 적용하며, macOS 26 계열의 scene 활성화 타이밍을 고려해 0.2초 지연 재적용을 수행한다.

## 버려야 할 이전 가정

아래 항목은 근거가 불충분하므로 원인으로 단정하지 않는다.

- Tahoe가 모든 서드파티 `NSStatusItem`을 기본 차단한다는 주장
- `ControlCenter`가 200ms 주기로 `VisibleCC = 0`을 강제한다는 주장
- 노치 영역 때문에 시스템 승인을 받지 않은 아이콘이 자동 drop된다는 주장
- `defaults write com.apple.controlcenter "NSStatusItemSpacing"`와 `killall ControlCenter`를 앱 해결책으로 사용하는 방식

이 항목들은 사용자 환경에서 관측될 수 있는 증상과 혼동하기 쉽지만, 배포 가능한 앱 수정안이나 공식 진단 결론으로 사용하면 안 된다.

## 현재 구현 점검

### Status item 생성

현재 구현:

- `Sources/MacSnow/StatusMenuController.swift`
- `private let statusItem = NSStatusBar.system.statusItem(withLength: 22)`
- `statusItem.autosaveName = "local.macsnow.statusItem"`
- `statusItem.menu = menu`

평가:

- status item을 강하게 보관하고 있으므로 즉시 해제되는 문제는 아니다.
- `autosaveName`은 이미 적용되어 있다.
- 길이 `22`는 작은 아이콘에는 적절하지만, 필요하면 `NSStatusItem.squareLength`와 비교 테스트한다.

### 아이콘 이미지

현재 구현:

- `NSImage(systemSymbolName: "snowflake", accessibilityDescription: "MacSnow")`
- `image.isTemplate = true`
- `button.imagePosition = .imageOnly`
- `button.imageScaling = .scaleProportionallyDown`
- 0.2초 지연 후 재적용

평가:

- template image, accessibility description, image-only 표시 방식은 적절하다.
- 지연 재적용은 Tahoe의 scene/status item 초기화 타이밍 회피책으로 유지한다.
- 단, 앱 시작 직후 짧은 시간 동안 빈 status item이 될 수 있으므로 즉시 적용과 지연 재적용을 모두 수행하는 구조가 더 낫다.

### 앱 실행 형태

현재 구현:

- `Scripts/build_app_bundle.sh`가 `LSUIElement = true`를 포함한 앱 번들을 만든다.
- `Sources/MacSnow/main.swift`가 `.accessory` activation policy를 설정한다.
- `AppDelegate.applicationDidFinishLaunching`에서는 바로 시작하지 않고, run loop 진입 후 `DispatchQueue.main.async`에서 manager를 시작한다.

평가:

- 메뉴 막대 앱 형태에 맞는 설정이다.
- SwiftPM 바이너리를 직접 실행하는 경우와 `.app` 번들로 실행하는 경우의 동작이 다를 수 있으므로, Tahoe 검증은 반드시 `dist/MacSnow.app` 기준으로 수행한다.

## 실행 순서

### 1. 코드 기준선 확인

목표:

- status item 생성, 이미지 적용, activation policy, `LSUIElement`가 모두 현재 코드에 반영되어 있는지 확인한다.

명령:

```bash
swift build
bash Scripts/build_app_bundle.sh
```

통과 기준:

- 빌드가 성공한다.
- `dist/MacSnow.app/Contents/Info.plist`에 `LSUIElement`가 `true`로 존재한다.
- 앱 실행 후 로그에 `statusItem`과 `button`이 생성된 기록이 남는다.
- 0.2초 지연 재적용 로그가 남는다.

### 2. 사용자 설정 확인

목표:

- macOS Tahoe의 앱별 메뉴 막대 표시 설정 때문에 숨겨진 상태인지 확인한다.

절차:

1. `dist/MacSnow.app`을 실행한다.
2. 시스템 설정을 연다.
3. `Menu Bar` 섹션으로 이동한다.
4. `Allow in the Menu Bar` 목록에서 `MacSnow`가 보이면 표시하도록 선택한다.
5. MacSnow를 종료 후 다시 실행한다.

통과 기준:

- 설정 변경 후 MacSnow 아이콘이 메뉴 막대에 표시된다.
- 아이콘 클릭 시 MacSnow 메뉴가 열린다.

주의:

- 이 설정은 사용자가 제어하는 시스템 UI다. 앱이 이를 강제로 변경하려고 해서는 안 된다.
- 앱 내부에서는 “시스템 설정에서 MacSnow 메뉴 막대 항목 표시를 켜세요” 수준의 안내만 제공한다.

### 3. 번들 실행과 SwiftPM 직접 실행 비교

목표:

- `.app` 번들 실행과 `swift run MacSnow` 실행의 차이를 분리한다.

절차:

```bash
swift run MacSnow
open dist/MacSnow.app
```

관찰 항목:

- 두 실행 방식 모두 아이콘이 보이는지
- 한쪽만 보인다면 `Bundle.main.bundlePath`, `CFBundleIdentifier`, `LSUIElement`, activation policy 로그가 어떻게 다른지

판정:

- 릴리스 기준은 `dist/MacSnow.app`이다.
- `swift run`에서만 실패하면 개발 실행 방식의 제약으로 분류하고 배포 차단 이슈로 보지 않는다.

### 4. 코드 보강 후보

아래 순서로만 적용한다. 한 번에 여러 가설을 섞지 않는다.

1. status item 버튼 이미지 즉시 적용 헬퍼를 만들고, 같은 헬퍼를 지연 재적용에서도 사용한다.
2. `statusItem.length`를 `22`와 `NSStatusItem.squareLength`로 비교 테스트한다.
3. 앱 시작 시 status item 생성 순서를 `.accessory` 설정 전후로 비교 테스트한다.
4. `NSApplication.didBecomeActiveNotification`에서 버튼 이미지가 사라졌는지 로그만 남긴다. 자동 복구는 재현 후 추가한다.

### 5. 금지 조치

아래 작업은 기본 실행 계획에 포함하지 않는다.

- `defaults write com.apple.controlcenter ...`로 사용자 전역 메뉴 막대 spacing 변경
- `killall ControlCenter` 실행
- private API, undocumented preference, 시스템 DB 직접 수정
- 검증 없는 “권한 획득 여부” 함수 구현
- `hasMenuBarPermission() -> true` 같은 더미 체크를 제품 코드에 추가

## 사용자 안내 문구 초안

MacSnow 아이콘이 메뉴 막대에 보이지 않는 경우:

> macOS 설정에서 MacSnow의 메뉴 막대 항목이 숨겨져 있을 수 있습니다. 시스템 설정 > Menu Bar > Allow in the Menu Bar에서 MacSnow 표시를 켠 뒤 앱을 다시 실행해 주세요.

이 안내는 앱 내부 alert 또는 README troubleshooting 섹션에 둘 수 있다.

## 결론

MacSnow의 Tahoe status bar 문제는 “보안 권한 차단”으로 단정하지 않는다. 현재 가장 신뢰할 수 있는 접근은 다음 순서다.

1. 앱 번들 기준으로 실행한다.
2. status item이 생성되고 이미지가 적용되는지 로그로 확인한다.
3. Tahoe `Menu Bar > Allow in the Menu Bar` 설정을 확인한다.
4. 필요한 경우에만 작은 코드 보강을 한 번에 하나씩 적용한다.
