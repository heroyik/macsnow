# Distribution Checklist

## Objective

Track the work needed to turn the SwiftPM prototype into a signed, distributable macOS app.

## Checklist

| Area | Task | Status |
| --- | --- | --- |
| Bundle | Add an app bundle target or Xcode project wrapper around the SwiftPM executable. | Started with `Scripts/build_app_bundle.sh` |
| Version | Keep `AppVersion.current`, `VERSION`, and release notes aligned before each release. | Pending |
| Identifier | Set a stable bundle identifier before signing. | Pending |
| Entitlements | Keep the app unsandboxed unless distribution requirements change. | Pending |
| Signing | Configure Developer ID Application signing for direct distribution. | Pending |
| Notarization | Notarize the app and staple the ticket. | Pending |
| Packaging | Produce a `.dmg` or `.zip` artifact with the app bundle. | Pending |
| Smoke Test | Launch the packaged app on a clean macOS user account. | Pending |
| Privacy | Reconfirm that window scanning only reads layout metadata. | Pending |
| Release Notes | Document known limitations around fullscreen apps, Stage Manager, and true particle physics. | Pending |

## Current Prototype State

The project is still primarily a SwiftPM executable prototype. `swift run MacSnow` remains the development launch path.

`Scripts/build_app_bundle.sh` creates a local unsigned `dist/MacSnow.app` bundle for smoke testing. Signing, notarization, and installer packaging remain pending.
