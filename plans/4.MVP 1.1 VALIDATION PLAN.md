# MVP 1.1 Validation Plan

## Objective

Define a repeatable manual validation routine for the current Xsnow prototype before adding more physics or packaging work.

## Preconditions

- Build from a clean terminal with `swift build`.
- Launch with `swift run Xsnow`.
- Keep Activity Monitor open for CPU and memory observation.
- Keep at least two ordinary app windows open, such as Finder and Safari.

## Validation Matrix

| --- | --- | --- | --- |

| Window Scanner | Move or open visible app windows while accumulation is enabled. | Debug `windows` and `edges` counts update. | Pending |

## Debug Menu Interpretation

- `overlays`: number of active overlay windows, expected to match connected `NSScreen` count.
- `active`: number of scenes currently rendering snow.
- `windows`: number of scanned visible third-party windows used by the scanner.
- `edges`: number of per-display top edges sent to SpriteKit scenes.
- Density title: currently selected global snow density.

## Known Manual Risks

- Fullscreen apps, Stage Manager, and multi-Space behavior still require manual validation with the Fullscreen Friendly level.
- `CGWindowListCopyWindowInfo` window bounds can include windows that are partly offscreen or covered by other windows.
- Current accumulation is a lightweight visualization layer, not true per-particle snow physics.
