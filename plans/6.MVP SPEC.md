# MVP Spec

## Product

Xsnow for macOS is a zero-dependency AppKit/SpriteKit menu bar prototype that renders non-blocking snow overlays across connected macOS displays.

Current MVP version: `1.20.0`

## Goals

- Run as a lightweight menu bar utility without a Dock icon.
- Create one transparent, click-through overlay per connected display.
- Render falling snow through SpriteKit.
- Persist user-facing controls across app launches.
- Track visible third-party window bounds and visualize snow accumulation on window top edges.
- Keep window scanning and visual node counts bounded for a menu bar prototype.
- Provide enough debug and validation hooks to verify behavior before packaging.

## Non-Goals

- Realistic large-scale accumulation is not implemented yet.
- App bundle packaging, signing, notarization, auto-update, and Mac App Store distribution are not implemented.
- Fullscreen, Stage Manager, and Spaces behavior have an initial Fullscreen Friendly level but still require manual validation.
- Automated tests are not available yet in the current local Swift toolchain setup.

## Runtime Architecture

- `main.swift`: starts `NSApplication` in accessory mode.
- `AppDelegate`: creates and starts `XsnowManager`.
- `XsnowManager`: central orchestrator for settings, menu actions, display overlays, window scanning, and sleep/wake lifecycle.
- `StatusMenuController`: owns the menu bar item, menus, and callbacks.
- `DisplayDetector`: observes display topology changes.
- `DisplayController`: owns one `OverlayWindow`, `SKView`, and `SnowScene` per display.
- `OverlayWindow`: transparent, borderless, click-through window.
- `SnowScene`: SpriteKit scene for snow emitter, bottom drift, seasonal objects, bounded physics flakes, collision edges, accumulation, spill/collapse fragments, and edge debug guides.
- `WindowLayoutScanner`: samples visible third-party window bounds with `CGWindowListCopyWindowInfo`.
- `WindowCoordinateMapper`: converts CoreGraphics desktop coordinates into per-display SpriteKit edge coordinates.
- `SettingsStore`: persists global and per-display settings through `UserDefaults`.
- `AppVersion`: single in-code version source for menu fallback.

## Menu Controls

- `Start Snow` / `Stop Snow`: toggles global snow rendering.
- `Density`: `Low`, `Normal`, `High`.
- `Wind`: `Off`, `Light`, `Medium`; default is `Light`.
- `Window Accumulation`: enables or disables window top-edge accumulation and scanning.
- `Small Flake Accumulation`: controls whether physics snowflake contacts increase accumulation height.
- `Spill When Full`: emits detailed falling snow fragments when buildup reaches the configured threshold.
- `Accumulation Rate`: `Low`, `Normal`, `High`.
- `Accumulation Style`: `Soft`, `Layered`, `Detailed`.
- `Clear Accumulation`: clears current buildup and falling collapse fragments without disabling future buildup.
- `Overlay Level`: `Normal`, `Fullscreen Friendly`, `Above Menu`.
- `Physics Quality`: `Low`, `Balanced`, `High`.
- `Show Edge Debug`: toggles red top-edge guide lines.
- `Displays`: toggles snow per connected display.
- `Debug`: shows overlay count, active scene count, scanned window count, collision edge count, and density.
- `Version`: displays the current version.
- `Quit Xsnow`: terminates the app.

## Display Behavior

- Uses `NSScreen.screens` to build overlay windows.
- Rebuilds overlays after `NSApplication.didChangeScreenParametersNotification`.
- Uses `DisplayIdentity` based on `NSScreenNumber` when available, with frame-based fallback.
- Per-display menu state is regenerated after display changes.

## Overlay Behavior

- Borderless `NSWindow`.
- Transparent background.
- `ignoresMouseEvents = true` for click-through interaction.
- `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary, .stationary]`.
- Overlay level is configurable:
  - `Normal`: `.mainMenu`
  - `Fullscreen Friendly`: `.floating`
  - `Above Menu`: `.mainMenu + 1`

## Snow Rendering

- Uses one `SKEmitterNode` per display scene.
- Renders a bottom snow drift layer with small clumps to ground the scene.
- Renders lightweight seasonal objects near the bottom edge.
- Uses a bounded `SKShapeNode` physics flake layer for prototype collision feedback.
- Physics flakes collide with tracked window top-edge physics bodies.
- Physics flake contacts increment the matching edge's accumulation height.
- `Physics Quality` controls maximum physics flakes, spawn interval multiplier, and contact-driven accumulation growth.
- Physics flakes are capped per scene and removed when they leave the visible region.
- Snow density controls particle birth rate.
- Wind controls x acceleration.
- Saved wind selection is restored on relaunch.
- Global snow toggle pauses/resumes scenes and birth rate.
- Sleep/wake pauses rendering and resumes according to stored settings.

## Window Scanning

- Uses `CGWindowListCopyWindowInfo([.optionOnScreenOnly, .excludeDesktopElements], kCGNullWindowID)`.
- Excludes the Xsnow process itself.
- Excludes windows smaller than `100x100`.
- Caps scanned windows at 20 in the scanner.
- Scanner runs only when global snow and window accumulation are both enabled.
- Scanner stops and clears tracking when global snow or window accumulation is disabled.
- Scanner also stops on sleep.

## Coordinate Mapping

- CoreGraphics window bounds are interpreted in desktop coordinates.
- `WindowCoordinateMapper` converts bounds into AppKit-style coordinates using the union desktop frame.
- Each display receives only visible intersecting window top edges.
- Edges narrower than 40 points are ignored.
- Per-scene collision edge count is capped at 32.

## Accumulation and Collapse

- Accumulation remains a lightweight visual layer; bounded physics flake contacts now contribute to accumulation height.
- Physics flake contact growth can be toggled separately through `Small Flake Accumulation`.
- Stable window edge keys accumulate snow height over repeated scans.
- `Accumulation Rate` controls growth speed and maximum height.
- Accumulation rendering supports Soft, Layered, and Detailed `SKShapeNode` drift styles.
- When `Spill When Full` is enabled, high buildup emits detailed falling sheets and small snow particles, then reduces local buildup.
- When a tracked edge disappears or moves, existing accumulation emits short falling collapse fragments.
- Collapse fragments are capped at 64 nodes per scene.
- `Clear Accumulation` removes current buildup and collapse fragments.
