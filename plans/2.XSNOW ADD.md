M5 맥북에어 환경에서 제로 디펜던시(Zero Dependency)와 초경량·고성능을 달성하기 위한 \*\*Xsnow for macOS 단계별 상세 설계서(Architecture Design Document)\*\*입니다.  
ENTJ 스타일의 직관적이고 컴팩트한 모듈 구조로 설계했습니다.

## **\[단계 2\] 모듈별 상세 설계 (3단계 구현 로드맵)**

### **1단계: 윈도우 인프라 및 다중 모니터 토폴로지 구축**

* **목표:** 시스템 트레이 상주 및 다중 모니터 영역을 100% 커버하는 투명 윈도우 동적 배치  
* **컴포넌트 설계:**  
  * XsnowManager: NSStatusItem을 생성하여 메뉴바에 아이콘 및 슬라이더 제어 UI 팝업 연결.  
  * DisplayDetector: NotificationCenter에 .NSApplicationDidChangeScreenParameters 리스너를 등록. 디스플레이 변경 시 기존 오버레이 윈도우를 완전히 해제(orderOut)하고 NSScreen.screens 정보를 기반으로 재구축.  
  * OverlayWindow: NSWindow를 서브클래싱하여 구현.  
    * styleMask \= .borderless  
    * backingType \= .buffered  
    * level \= .mainMenu \+ 1 (풀스크린 전환 시 레이어 스왑 로직 포함)  
    * ignoresMouseEvents \= true (Click-Through 완벽 구현)

### **2단계: 윈도우 레이아웃 스캐너 및 좌표 변환**

* **목표:** 타사 앱의 창 상단 테두리를 정확히 추적하되 시스템 부하 최소화  
* **컴포넌트 설계:**  
  * WindowLayoutScanner: DispatchSourceTimer를 이용해 백그라운드 큐에서 500ms 주기로 동작.  
  * **핵심 로직:**  
    1. CGWindowListCopyWindowInfo(\[.optionOnScreenOnly, .excludeDesktopElements\], kCGNullWindowID) 호출.  
    2. 오너 앱이 본인 앱(Xsnow)이거나 메인 메뉴바인 경우는 필터링 제외.  
    3. 딕셔너리에서 kCGWindowBounds를 추출하여 CGRect 배열 생성.  
    4. **좌표계 변환:** CoreGraphics 좌표계(좌상단 0,0)를 SpriteKit/AppKit 좌표계(우상단 혹은 개별 모니터 좌하단 0,0)로 정교하게 매핑 연산 진행.  
    5. 수집된 Top Edge 정보들(x1, x2, y)을 전역 NotificationCenter를 통해 각 모니터의 SnowPhysicsEngine으로 브로드캐스팅.

### **3단계: SpriteKit 기반 고성능 2D 물리 엔진 설계**

* **목표:** M5 GPU 가속을 활용한 입자 충돌 및 연출 처리 (배터리 소모 차단)  
* **컴포넌트 설계:**  
  * SnowPhysicsEngine (SKScene):  
    * physicsWorld.gravity \= CGVector(dx: 0.0, dy: \-1.0) (기본 중력값 설정)  
  * **눈송이 시스템:** SKEmitterNode를 활용하며 입자 텍스처는 2x2, 3x3, 4x4 크기의 단순 화이트 픽셀 맵을 생성하여 바인딩. 원근감을 위해 alpha, speed 값을 난수 분산 처리.  
  * **동적 물리 벽 (Dynamic Collision Collision):**  
    * 스캐너로부터 창 좌표 배열을 수신하면, 기존의 물리 바디(SKNode들의 physics())를 전부 제거.  
    * 전달받은 창 상단 라인에 매칭되는 SKNode를 만들고, SKPhysicsBody(edgeLoopFrom: ...)를 부여.  
    * 창이 이동했을 경우 기존 고정(Static) 상태로 멈춰있던 눈송이 입자들과의 물리적 접촉면이 사라지므로 자연스럽게 하강이 재개됨.  
  * **산타 & 썰매 시퀀스:**  
    * SKAction.sequence를 기반으로 무작위 타이밍 오프셋을 계산하여 화면 외부 \-x에서 \+x로 등속 이동 애니메이션 수행.  
    * 바람의 세기 세팅에 따라 SKAction.speed 계수를 실시간 연동.

## **\[단계 3\] 데이터 및 제어 흐름 설계 (통합 vs 개별 모니터)**

환경 설정 값 변경 시 레이턴시 없는 동기화를 위해 \*\*구독 모델(Publish-Subscribe)\*\*을 적용합니다.

### **데이터 구조 체계 (User Preferences)**

`struct XsnowDisplaySettings: Codable {`  
    `var isEnabled: Bool = true`  
    `var snowDensity: Float = 0.5 // 0.0 ~ 1.0`  
    `var windStrength: Float = 0.2`  
    `var santaFrequency: Float = 0.3`  
`}`

`struct XsnowGlobalConfig: Codable {`  
    `var globalSettings: XsnowDisplaySettings = XsnowDisplaySettings()`  
    `var perDisplaySettings: [String: XsnowDisplaySettings] = [:] // Key: Display ID`  
    `var useGlobalOnly: Bool = true`  
`}`

### **흐름 제어 로직**

1. **통합 제어 모드일 때:**  
   * 메뉴바 UI 슬라이더 조작 \-\> globalSettings 값 업데이트 \-\> 전역 이벤트 발행 \-\> 모든 활성화된 SnowPhysicsEngine 인스턴스가 동일한 속성(emitter.birthRate, gravity.dx) 반영.  
2. **모니터별 제어 모드일 때:**  
   * 설정 창에서 특정 모니터 ID(NSScreen 고유 번호)의 슬라이더 조작 \-\> perDisplaySettings\[UUID\] 값만 수정 \-\> 해당 UUID를 소유한 DisplayController 인스턴스 내부의 SKScene 속성만 단독 수정.

## **\[단계 4\] 예외 처리 및 엣지 케이스 대응 설계**

| 엣지 케이스 | 현상 | 해결 설계 방안 |
| :---- | :---- | :---- |
| **맥북 수면 및 디스플레이 오프** | 백그라운드 렌더링 지속 시 배터리 드레인 발생 | NSWorkspace.shared.notificationCenter를 구독하여 .didSleep 이벤트 수신 즉시 모든 SKScene.isPaused \= true 처리. .didWake 발생 시 재개. |
| **타사 앱 전체 화면 (Full Screen)** | 가상 데스크톱 스페이스 이동 시 눈 오버레이가 가려지거나 엉뚱한 스페이스에 고정됨 | 윈도우 생성 시 collectionBehavior \= \[.canJoinAllSpaces, .fullScreenAuxiliary\] 속성을 명시하여 스테이지 매니저 및 전체 화면 스페이스 전환 시에도 오버레이 창이 상시 투명 추적하도록 격리. |

2. **\[구현 2단계\]** CGWindowListCopyWindowInfo 기반의 창 상단 엣지 추출 및 스케일 변환 유틸리티 클래스 구현하기

## **\[MVP v0.1\] 구현 아키텍처 범위**

MVP v0.1은 전체 물리 엔진을 완성하기 전에, 메뉴바 앱과 다중 모니터 투명 오버레이가 macOS에서 안정적으로 동작하는지 검증하는 단계입니다. 창 충돌, 산타 애니메이션, 바람 효과, 모니터별 상세 설정은 후속 버전으로 분리합니다.

### **1\. v0.1 컴포넌트 구성**

`+-------------------------------------------------------+`  
`|             XsnowManager (App Delegate / Orchestrator) |`  
`+-------------------------------------------------------+`  
`        | 메뉴바 제어                 | 디스플레이 변경 감지`  
`        v                            v`  
`+------------------------+   +------------------------+`  
`| StatusMenuController   |   | DisplayDetector        |`  
`| - NSStatusItem         |   | - NSScreen.screens     |`  
`| - Start/Stop Toggle    |   | - screen change notify |`  
`+------------------------+   +------------------------+`  
`                  |          |`  
`                  v          v`  
`+-------------------------------------------------------+`  
`| DisplayController (one per NSScreen)                  |`  
`| - OverlayWindow: transparent, borderless, click-through |`  
`| - SnowScene: simple SpriteKit falling snow             |`  
`+-------------------------------------------------------+`

### **2\. v0.1 구현 순서**

1. **앱 셸 구성:** Dock에 표시되지 않는 menu bar app으로 시작하고 `NSStatusItem` 메뉴를 만든다.
2. **디스플레이 감지:** `NSScreen.screens`를 기준으로 현재 연결된 모니터 목록을 읽고, 화면 변경 알림을 구독한다.
3. **오버레이 생성:** 각 `NSScreen.frame`에 맞는 borderless `NSWindow`를 만들고 `isOpaque = false`, `backgroundColor = .clear`, `ignoresMouseEvents = true`를 적용한다.
4. **SpriteKit 연결:** 각 오버레이에 `SKView`를 붙이고, 단순 `SnowScene`을 표시한다.
5. **전역 on/off:** 메뉴바 toggle 상태에 따라 모든 `SnowScene.isPaused` 또는 emitter birth rate를 제어한다.
6. **수면/깨우기 처리:** `NSWorkspace` sleep/wake 알림에서 렌더링을 일시정지/재개한다.

### **3\. v0.1에서 제외되는 모듈**

* `WindowLayoutScanner`: v0.1에서는 `CGWindowListCopyWindowInfo`를 호출하지 않는다.
* 동적 물리 벽 및 창 상단 충돌 처리
* 창 이동 감지 및 눈 붕괴 연출
* 모니터별 설정 저장 모델
* 산타, 루돌프, 트리, 북극곰 등 시즌 오브젝트
* 바람 알고리즘 및 고급 입자 제어

### **4\. v0.1 완료 기준**

* 앱 실행 시 모든 연결 디스플레이에 투명 오버레이가 생성된다.
* 오버레이 위에서도 하위 앱 클릭과 드래그가 정상 동작한다.
* 메뉴바에서 눈 효과를 켜고 끌 수 있다.
* 수면/깨우기 이후 오버레이와 눈 효과가 재개된다.
* 기본 밀도에서 체감상 부드럽게 동작하며, 구현 후 Instruments 또는 Activity Monitor로 CPU/RAM 사용량을 기록한다.

## **\[MVP v0.2\] 구현 아키텍처 범위**

MVP v0.2는 v0.1의 오버레이/렌더링 구조 위에 설정 모델과 모니터별 제어를 추가합니다. 창 좌표 스캐너와 충돌 물리는 아직 도입하지 않고, 사용자 제어와 상태 복원 품질을 먼저 안정화합니다.

### **1\. v0.2 추가 컴포넌트**

`+-------------------------------------------------------+`  
`|                  XsnowManager                         |`  
`+-------------------------------------------------------+`  
`        | 설정 읽기/쓰기          | 메뉴 이벤트 반영`  
`        v                         v`  
`+------------------------+   +------------------------+`  
`| SettingsStore          |   | StatusMenuController   |`  
`| - UserDefaults         |   | - density menu         |`  
`| - Codable config       |   | - per-display toggles  |`  
`+------------------------+   +------------------------+`  
`        |                         |`  
`        v                         v`  
`+-------------------------------------------------------+`  
`| DisplayController                                      |`  
`| - apply(display settings)                              |`  
`| - pause/resume per monitor                             |`  
`| - update SnowScene density/wind                        |`  
`+-------------------------------------------------------+`

### **2\. 데이터 모델**

`struct XsnowDisplaySettings: Codable {`  
`    var isEnabled: Bool`  
`}`  

`enum SnowDensity: String, Codable {`  
`    case low`  
`    case normal`  
`    case high`  
`}`  

`struct XsnowGlobalSettings: Codable {`  
`    var isSnowEnabled: Bool`  
`    var density: SnowDensity`  
`    var windStrength: Double`  
`    var perDisplay: [String: XsnowDisplaySettings]`  
`}`  

모니터 식별자는 `NSScreen.deviceDescription[NSDeviceDescriptionKey("NSScreenNumber")]` 값을 우선 사용하고, 값이 없으면 화면 frame 기반 fallback key를 사용합니다.

### **3\. v0.2 구현 순서**

1. **SettingsStore 추가:** `UserDefaults`에 `XsnowGlobalSettings`를 JSON으로 저장/복원한다.
2. **설정 적용 경로 정리:** `XsnowManager`가 앱 시작 시 설정을 읽고 모든 `DisplayController`에 적용한다.
3. **밀도 메뉴 추가:** `Low`, `Normal`, `High` 메뉴를 만들고 `SnowScene`의 birth rate를 즉시 갱신한다.
4. **모니터별 토글 추가:** 현재 연결된 디스플레이 목록을 메뉴에 표시하고, 선택한 모니터만 pause/resume한다.
5. **바람 값 적용:** `SnowScene`의 `xAcceleration`을 설정값에 따라 조정한다.
6. **디버그 메뉴 추가:** 오버레이 수, 활성 scene 수, 현재 density 값을 표시한다.

### **4\. v0.2 완료 기준**

* 전역 on/off, density, wind 설정이 앱 재실행 후 복원된다.
* 모니터별 enable/disable이 현재 연결된 디스플레이에만 정확히 적용된다.
* 디스플레이 변경 후 메뉴 목록과 오버레이 목록이 다시 동기화된다.
* `swift build`가 성공하고, 수동 검증 체크리스트를 README에 갱신한다.

## **\[MVP v0.3\] 구현 아키텍처 범위**

MVP v0.3은 창 충돌 물리의 전 단계로, 창 좌표 수집 파이프라인을 추가합니다. 수집된 좌표는 아직 `SnowScene`에 물리 바디로 적용하지 않고, 디버그 정보로만 노출합니다.

### **1\. v0.3 추가 컴포넌트**

`+-------------------------------------------------------+`  
`|                  XsnowManager                         |`  
`+-------------------------------------------------------+`  
`        | 500ms snapshot 수신`  
`        v`  
`+------------------------+`  
`| WindowLayoutScanner    |`  
`| - CGWindowList         |`  
`| - bounds filtering     |`  
`| - max 20 windows       |`  
`+------------------------+`

### **2\. 스캔 정책**

* `CGWindowListCopyWindowInfo([.optionOnScreenOnly, .excludeDesktopElements], kCGNullWindowID)`를 사용한다.
* 앱 본인 창, 데스크톱 요소, 100x100 미만 창은 제외한다.
* 스캔 결과는 `WindowSnapshot(ownerName, bounds)` 배열로 유지한다.
* MVP v0.3에서는 최대 20개까지만 사용한다.

### **3\. 후속 설계 과제**

* CoreGraphics 좌상단 기준 bounds를 각 `NSScreen`/SpriteKit scene 좌표로 변환한다.
* 모니터별로 포함되는 창 top edge만 분리한다.
* v0.4에서 top edge collision body를 `SnowScene`에 적용한다.
