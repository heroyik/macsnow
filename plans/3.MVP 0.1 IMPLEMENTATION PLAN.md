# MVP 0.1 Implementation Plan

## Objective

Build the first runnable MacSnow prototype that proves the core platform behavior: a menu bar app creates transparent click-through overlays on every connected display and renders a simple SpriteKit snow effect.

MVP 0.1 does not include third-party window collision, snow accumulation, Santa animation, wind physics, per-display settings, or distribution packaging.

## Target Scope

### Included

- Menu bar only app using `NSStatusItem`
- No Dock icon
- Multi-display detection through `NSScreen.screens`
- One transparent overlay `NSWindow` per display
- Click-through behavior with `ignoresMouseEvents = true`
- One `SKView` and basic `SnowScene` per overlay
- Global enable/disable menu item
- Sleep/wake pause and resume handling

### Excluded

- `CGWindowListCopyWindowInfo`
- Collision against third-party windows
- Snow buildup and collapse physics
- Per-monitor configuration UI
- Persistent settings
- Seasonal objects and wind effects
- Notarization, signing, and Mac App Store packaging

## Implementation Phases

### Phase 1: Xcode Project Bootstrap

Create a macOS AppKit project named `MacSnow`.

Key setup:

- Language: Swift
- Interface: AppKit, no SwiftUI dependency required for MVP
- App category: menu bar utility
- `LSUIElement = true` in `Info.plist` to hide Dock icon
- Deployment target: define explicitly before implementation begins

Deliverable:

- App launches without Dock icon
- Menu bar item appears
- Quit action works

## Phase 2: Menu Bar Controller

Implement `StatusMenuController`.

Responsibilities:

- Create `NSStatusItem`
- Show simple menu
- Provide `Start Snow`, `Stop Snow`, and `Quit`
- Forward state changes to `MacSnowManager`

Acceptance criteria:

- User can enable and disable snow globally
- Menu state reflects current running state

## Phase 3: Display Detection

Implement `DisplayDetector`.

Responsibilities:

- Read `NSScreen.screens`
- Expose current display list
- Subscribe to `NSApplication.didChangeScreenParametersNotification`
- Notify `MacSnowManager` when displays change

Acceptance criteria:

- Connecting or disconnecting a display rebuilds overlays without restarting the app
- Existing overlays are released before new ones are created

## Phase 4: Transparent Overlay Windows

Implement `DisplayController` and `OverlayWindow`.

Responsibilities:

- Create one borderless window per display
- Match each window to `NSScreen.frame`
- Use transparent background
- Set `ignoresMouseEvents = true`
- Keep overlays above normal app windows without blocking interaction

Acceptance criteria:

- Snow overlay covers each display
- Finder, Safari, and Xcode remain clickable through the overlay
- Overlay does not appear in the Dock or app switcher

## Phase 5: Basic SpriteKit Snow Scene

Implement `SnowScene`.

Responsibilities:

- Render simple falling snow particles
- Avoid external image assets for MVP; use generated or simple SpriteKit particles
- Support pause/resume
- Expose basic density constant in code

Acceptance criteria:

- Snow is visible on each display
- Animation is smooth at default density
- Stop command pauses or removes rendering immediately

## Phase 6: Sleep/Wake Lifecycle

Add lifecycle handling in `MacSnowManager`.

Responsibilities:

- Subscribe to `NSWorkspace.willSleepNotification`
- Subscribe to `NSWorkspace.didWakeNotification`
- Pause scenes before sleep
- Resume scenes after wake if snow was enabled

Acceptance criteria:

- App does not continue rendering during sleep transition
- Snow resumes after wake without relaunch

## Phase 7: Manual Validation

Run manual checks before marking MVP complete.

Checklist:

- Launch app
- Confirm menu bar item appears
- Confirm Dock icon is hidden
- Confirm snow appears on all connected displays
- Confirm click-through behavior in Finder, Safari, and Xcode
- Toggle snow off and on
- Connect or disconnect an external display
- Sleep and wake the Mac
- Record CPU and memory usage from Activity Monitor

## Risks and Follow-Ups

- Window level behavior may need tuning for full-screen apps, Stage Manager, and multiple Spaces.
- `NSScreen.frame` coordinate handling may need correction for displays arranged above or left of the main display.
- Performance targets should be measured after the first SpriteKit implementation rather than assumed.
- Persistent preferences should be added in MVP 0.2 after the global toggle behavior is stable.

## Definition of Done

MVP 0.1 is complete when a user can launch the app, see falling snow on every connected display, continue using underlying apps normally, toggle the effect from the menu bar, and recover correctly from display changes and sleep/wake.
